Nos Technologies

Solutions Web

Oracle ATG Web Commerce

Oracle ATG Web Commerce est une solution e-commerce qui addresse les besoins d'une boutique en ligne :

  • Boutique e-commerce : gestion du catalogue, des prix, des promotions et du panier. Depuis la version 11, la solution supporte les boutiques multi-sites (plusieurs marques, plusieurs pays, plusieurs langues).
  • Recherche aidée : personnalisation des recherches et de la pertinence des résultats avec le moteur de recherche Oracle Endeca. La navigation et les suggestions de "cross selling" ou de "up selling" sur votre site est guidée par le moteur. Cette fonctionnalité est globalement appelée Oracle Experience Manager.
  • Traitement middle office des commandes : Customer Service Center est une application qui vous permet de gérer le traitement des commandes une fois le panier de l'utilisateur payé. Cet outil permet de gérer les retours, les échanges ou tout simplement la verification/validation de la commande avant traitement par votre ERP interne.

Harp JS

Notre site est construit avec Harp JS, un framework Javascript qui permet de générer des sites statiques avec un code plus dynamique que du HTML simple. Nous avons choisi ce produit car il fonctionne sur NodeJS, déjà installé sur nos plateformes, et qu'il supporte l'utilisation de templates Jade (langage simple et succint de templating de nouvelle génération).

Frameworks MVC

Nous utilisons des frameworks légers et performants pour développer des applications web rapidement. Que ce soit Sails JS pour NodeJS ou Play framework écrit en JAVA, nous privilégions la prise en main rapide et la facilité de développement pour réduire le temps de livraison des sites. Ces deux solutions présentent l'avantage d'utiliser un outil de "scaffolding" pour la création de nouveaux éléments techniques dans votre projet.

Outils d'integration continue

Jenkins

Outil incontournable de l'integration continue JAVA/J2EE qui peut aussi être utilisé pour d'autres technologies, Jenkins permet le lancement automatique de taches de compilation et de livraison de code pour permettre la validation des modifications apportées par les équipes de développement.

Gestion opérationnelle de plateformes

Opscode Chef

Outil d'automatisation de la gestion des fichiers de configurations de serveur. L'avantage de Chef  par rapport à d'autres outils tels que Puppet ou Ansible c'est qu'il est capable de requêter un serveur centralisé et de configurer intelligement un nouveau noeud suivant son rôle. Il est par exemple possible que les configurations des serveurs web se mettent à jour automatiquement si Chef detecte que vous avez une nouvelle instance d'application qui a été instanciée.

Sensu

Sensu est un système de supervision conçu pour travailler avec des milliers de machines : l'utilisation de Rabbitmq pour les communications entre les noeuds et le serveur de supervision garantit une scalabilité et une réactivité du système. Ce choix est également cohérent avec le choix de Chef car les deux systèmes ont des pré-requis techniques similaires (Ruby, Rabbitmq).

Note : Nous utilisons Chef et Sensu pour nos propres serveurs internes même si pour le moment, leur nombre ne semble pas justifier ces outils. Nous voulons que nos collaborateurs soient immergés dans ces solutions même pour des installations internes.
